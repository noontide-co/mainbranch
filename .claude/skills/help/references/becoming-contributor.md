# Becoming a Contributor

Main Branch is built by its community. If you want to contribute back, here's how.

---

## What Contributors Do

Contributors help improve the engine (vip) for everyone:

- **Skills** - Build new skills or improve existing ones
- **Frameworks** - Add compliance frameworks, review lenses
- **Documentation** - Improve help content, fix gaps
- **Domain Rubrics** - Add templates for new business types
- **Testing** - Try things, report what works and what breaks

---

## Why Contribute?

### Give Back
You benefit from others' contributions. Pay it forward.

### Sharpen Your Skills
Building skills teaches you Claude Code deeply.

### Recognition
Contributors are credited in the community.

### Shape the Tool
Your use cases become part of the system.

---

## Getting Started

### 1. Use Main Branch Deeply First

Spend time with the system. Run skills. Build your reference. Understand how it works.

Contributors who use the tool daily build better additions.

### 2. Notice Gaps

As you work, you'll think "this should exist" or "this could be better."

That's your contribution waiting to happen.

### 3. Propose Before Building

Share your idea in Skool before investing time. Devon and the community can help you:
- Validate the need
- Avoid duplicate effort
- Suggest the right approach

### 4. Build It

Once validated, build your contribution:
- Skills follow the skill-creator pattern
- Documentation follows existing conventions
- Test thoroughly before submitting

### 5. Submit for Review

Share your work for feedback. Iterate based on input. Merge when ready.

## The Inner Circle

Active contributors may be invited to the Inner Circle:

- Direct access to Devon
- Early access to new features
- Input on roadmap decisions
- Recognition in the community

This isn't bought - it's earned through contribution quality.

---

## Technical Requirements

To contribute to vip, you need:

1. **Git basics** - Clone, branch, commit, pull request
2. **Skill understanding** - How SKILL.md files work
3. **Testing ability** - Verify your work before submitting
4. **Documentation skills** - Clear writing for other Claudes and humans

If you're not there yet, using Main Branch more will get you there.

---

## The Culture

Main Branch contributors:

- **Ship working code** - Tested, documented, reliable
- **Help each other** - Answer questions, review work
- **Iterate openly** - Share progress, accept feedback
- **Credit sources** - Acknowledge inspiration and collaboration
- **Maintain quality** - The bar is high because it affects everyone

---

## Starting Small

Don't try to build a massive skill on day one.

Good first contributions:
- Fix a typo in documentation
- Add a troubleshooting scenario you encountered
- Document a workflow that isn't captured
- Improve an explanation that confused you

Small, quality contributions build trust and experience.
