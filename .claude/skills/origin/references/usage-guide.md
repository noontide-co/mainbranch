# When to Use origin.md

Centralized guidance for how content skills should use origin.md. Origin content is powerful but should be used selectively — not every piece of content needs the founder's story.

---

## General Principle

**Use origin.md when the founder's identity, journey, or values add trust or authenticity.**

**Skip origin.md when the focus should be on the audience's problem, the mechanism, or practical value.**

---

## By Skill

### /ads

| Use origin.md | Skip origin.md |
|---------------|----------------|
| Deep ads that need credibility | Pattern interrupt ads (pure hook) |
| Authority plays ("After 17 years...") | UGC-style native ads (customer voice) |
| "Why I created this" angle | Mechanism-focused direct response |
| High-ticket trust building | Most one-liners (unless founder IS the hook) |
| Founder/creator angle explicitly | Testimonial ads (customer story, not founder) |

### /organic

| Use origin.md | Skip origin.md |
|---------------|----------------|
| Story-based scripts (personal narrative) | Educational/tips content |
| "Why I do this" or "how I got here" | Problem-solution scripts |
| Credibility intros ("After X years...") | Listicles and how-tos |
| Values-based messaging | Most carousels (unless story carousel) |
| Transformation framework (your journey) | Practical tutorials |

### /vsl

| Section | Origin Usage |
|---------|--------------|
| **Section 2 (Epiphany Bridge)** | Heavy — this IS the founder story section |
| Credibility mentions | Light touch where trust matters |
| "Why I created this" beat | Use if script includes this |
| Features / The Plan | Skip — focus on what they get |
| Proof sections | Skip — customer stories, not founder |
| CTA sections | Skip — their action, not your journey |
| Price anchoring | Skip — about value, not origin |

---

## Quick Decision Framework

Ask: **"Does the founder's story add trust here, or distract from the message?"**

- If the content is about **identity, values, or credibility** → use origin.md
- If the content is about **their problem, the solution, or practical steps** → skip it

---

## What to Pull from origin.md

When you DO use it, pull selectively:

| Need | Pull From |
|------|-----------|
| Credibility marker | Years of experience, background, credentials |
| Unique angle | What makes this founder different |
| Values | What they stand for, authenticity markers |
| The "why" | Why they created this, what drives them |
| Transformation | Their before/after journey |

Don't dump the whole origin story. Extract the piece that serves THIS content.
