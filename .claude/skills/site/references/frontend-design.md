# Frontend Design Principles

> **Applies to: Website (Next.js) shape only.** Minisite generation has its own design system in [`minisite-generation-system.md`](minisite-generation-system.md) — variance-first, no fixed typography or palette. Use this file when you're working in the legacy Next.js Website templates.

Design system for landing pages generated by `/site` Website shape. Adapted from Anthropic's frontend-design skill, rewritten for conversion-focused pages driven by Main Branch reference files.

---

## Core Philosophy

**Design is invisible infrastructure for the offer.** The visitor should remember the headline, the transformation, and the price — not the font, the animation, or the layout. Every design decision answers: does this make the offer clearer, more credible, or more urgent? If not, cut it.

---

## Design Discovery (Before Coding)

Read reference files in this order. Design flows FROM the brand, not toward it.

1. **voice.md** — Tone, personality, energy. Drives aesthetic direction.
2. **offer.md** — What they sell, price point, mechanism. Drives layout and CTA.
3. **audience.md** — Who buys, sophistication, what they respond to. Drives visual language.
4. **soul.md** — Mission, founder story. Adds depth to color and typography.

### Voice-to-Aesthetic Derivation

| Voice Trait | Typography | Color Temperature | Spacing | Motion | Corners |
|---|---|---|---|---|---|
| Warm, personal | Serif or humanist sans display | Warm (amber, terracotta, sage) | Generous | Gentle, slow reveals | Rounded (8-12px) |
| Direct, authoritative | Geometric sans or strong serif | High contrast (navy/white, black/gold) | Structured | Crisp, fast | Sharp (0-4px) |
| Playful, energetic | Personality fonts, large display | Bold primaries, unexpected combos | Dynamic, varied | Bouncy, spring physics | Mixed |
| Premium, refined | Thin-weight serif, wide spacing | Restrained (2-3 colors max) | Ample whitespace | Minimal, elegant | Subtle (2-4px) |
| Technical, expert | Monospace accents, clean sans | Muted palette + one signal color | Tight, dense | Precise, functional | Sharp (0-2px) |

## Anti-AI-Slop Standards

### Never Use

- **Fonts:** Inter, Roboto, Arial, Poppins, Helvetica as primary font
- **Colors:** Purple gradient on white background. Blue-to-purple gradients.
- **Layout:** Centered-everything. Symmetrical 3-card grids with icons. Hero with stock photo right, text left.
- **Components:** Cookie-cutter rounded cards with drop shadows. Generic pricing tables.
- **Copy:** "Unlock your potential." "Transform your [X]." "Join thousands of..." without specifics.
- **Spacing:** Identical padding on every section. No rhythm variation.

### Instead Do

- **Fonts:** Characterful choices derived from voice.md (see pairings below)
- **Colors:** Brand-derived. One dominant, one accent, rest neutral.
- **Layout:** Varied section rhythm. Asymmetric hero. Mixed full-bleed and contained sections.
- **Components:** Designed for the content. Powerful testimonial gets full-width, not a card.
- **Copy:** Pulled from offer.md and audience.md. Real language, real numbers.
- **Spacing:** Rhythm through contrast — dense proof, spacious benefits, tight urgency.

## Typography System

### Font Pairings by Brand Tone

**Warm / Personal:**
- Display: Fraunces, Lora, Playfair Display, Crimson Pro
- Body: Source Sans 3, Nunito Sans, DM Sans

**Direct / Authoritative:**
- Display: Space Grotesk, Outfit, Sora, Manrope
- Body: Work Sans, IBM Plex Sans, Geist

**Playful / Energetic:**
- Display: Bricolage Grotesque, Syne, Rubik
- Body: Plus Jakarta Sans, Lexend, Figtree

**Premium / Refined:**
- Display: Cormorant Garamond, EB Garamond, Instrument Serif
- Body: Jost, Libre Franklin, Outfit

**Technical / Expert:**
- Display: Space Grotesk, JetBrains Mono (display only)
- Body: IBM Plex Sans, Source Sans 3
- Mono: JetBrains Mono, IBM Plex Mono, Space Mono

```css
:root {

  --text-sm: clamp(0.875rem, 0.8rem + 0.35vw, 1rem);
  --text-base: clamp(1rem, 0.9rem + 0.5vw, 1.125rem);
  --text-lg: clamp(1.125rem, 1rem + 0.6vw, 1.25rem);
  --text-xl: clamp(1.25rem, 1.1rem + 0.75vw, 1.5rem);
  --text-2xl: clamp(1.5rem, 1.2rem + 1.5vw, 2rem);
  --text-3xl: clamp(1.875rem, 1.5rem + 1.9vw, 2.5rem);
  --text-4xl: clamp(2.25rem, 1.5rem + 3.75vw, 3.5rem);
  --text-hero: clamp(2.75rem, 1.75rem + 5vw, 4.5rem);

```

### Typography Rules

| Element | Scale | Weight | Line Height | Letter Spacing |
|---|---|---|---|---|
| Hero headline | `--text-hero` | 700-900 | 1.0-1.1 | -0.02em to -0.03em |
| Section heading | `--text-3xl` to `--text-4xl` | 600-700 | 1.1-1.2 | -0.01em |
| Body copy | `--text-base` to `--text-lg` | 400 | 1.6-1.7 | normal |
| Labels / eyebrows | `--text-xs` to `--text-sm` | 500-600 | 1.3 | 0.05em-0.1em (uppercase) |
| CTA button | `--text-base` to `--text-lg` | 600 | 1.0 | 0.02em |

## Color System

### Deriving Colors from Brand

| Positioning | Primary Direction | Accent Direction |
|---|---|---|
| Premium / luxury | Deep navy, charcoal, black | Gold, copper, warm white |
| Warm / approachable | Warm neutrals, cream | Terracotta, coral, amber |
| Bold / disruptive | Saturated primary | Contrasting complement |
| Technical / trustworthy | Cool grays, slate | Blue, teal, green |
| Creative / expressive | Confident choice | Unexpected complement |

### CSS Variable Template

```css
:root {
  --background: #0F1B27;
  --foreground: #F2F1EF;
  --accent: #D4AF37;
  --accent-rgb: 212, 175, 55;    /* For rgba() in box-shadows */
  --accent-hover: #E5C158;
  --muted: #8B9BAA;
  --card: #162029;
  --card-border: #2A3A47;
  --border: #2A3A47;
}
```

### Color Rules

- **60-30-10:** Background 60%, secondary 30%, accent 10%
- **CTA color is sacred.** Nothing else on the page uses the CTA color. When the user sees that color, it means: take action.
- **Dark vs light:** Dark backgrounds signal premium/technical. Light signals approachable/clean. Never pure white (#FFFFFF) — use #FAFAFA or warmer.
- **Contrast:** WCAG AA minimum (4.5:1 body, 3:1 large text).

## Motion System

### Hero Reveal (Mandatory)

Every landing page has an orchestrated hero entrance:

```css
.hero-content > * {
  opacity: 0;
  transform: translateY(30px);
}
.hero-content.revealed > * {
  opacity: 1;
  transform: translateY(0);
  transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}
.hero-content.revealed > *:nth-child(1) { transition-delay: 0.1s; }
.hero-content.revealed > *:nth-child(2) { transition-delay: 0.25s; }
.hero-content.revealed > *:nth-child(3) { transition-delay: 0.4s; }
.hero-content.revealed > *:nth-child(4) { transition-delay: 0.55s; }
.hero-content.revealed > *:nth-child(5) { transition-delay: 0.7s; }
```

For headline word-by-word animation, use GSAP + SplitType (both templates include these):

```typescript
import { gsap } from 'gsap'
import SplitType from 'split-type'

const split = new SplitType(headlineRef.current, { types: 'words' })
gsap.from(split.words, {
  y: 40, opacity: 0, duration: 0.6, stagger: 0.04, ease: 'power3.out', delay: 0.2,
})
```

### Scroll-Triggered Reveals

Use IntersectionObserver. CSS-only for simple reveals:

```typescript
export function useInView(threshold = 0.15) {
  const ref = useRef<HTMLDivElement>(null)
  const [inView, setInView] = useState(false)
  useEffect(() => {
    if (!ref.current) return
    const obs = new IntersectionObserver(
      ([e]) => { if (e.isIntersecting) setInView(true) },
      { threshold }
    )
    obs.observe(ref.current)
    return () => obs.disconnect()
  }, [])
  return { ref, inView }
}
```

### Motion Rules

| Rule | Rationale |
|---|---|
| Hero animation is mandatory | First impression, perceived quality |
| Total sequence: 2 seconds max | Longer feels sluggish |
| No entrance animation on CTAs | Must be instantly visible |
| Scroll reveals fire once | No re-animation on scroll back |
| Respect `prefers-reduced-motion` | Accessibility requirement |
| Stagger: 0.04s-0.08s per element | Sweet spot for reading rhythm |

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Section Ordering for Conversion

| # | Section | Purpose |
|---|---|---|
| 1 | Hero | Hook + promise + CTA |
| 2 | Trust strip | Logos/badges (if available) |
| 3 | Problem / Pain | Name the pain |
| 4 | Solution overview | Mechanism or demo |
| 5 | Process / How it works | Step-by-step clarity |
| 6 | Examples / Use cases | Relevance |
| 7 | Credibility / Proof | Testimonials, stats |
| 8 | Competitive positioning | Why this vs alternatives |
| 9 | Objection handling | Address fears |
| 10 | Pricing / Value stack | What you get, what it costs |
| 11 | FAQ | Remaining questions |
| 12 | Final CTA | Close with urgency |

Minimum viable: Hero + Problem + Solution + Proof + CTA (5 sections).

## What to Avoid

| Pattern | Why It Hurts |
|---|---|
| Custom cursors | Accessibility nightmare, mobile-irrelevant |
| Scroll-jacking | Users lose control, rage-quit |
| Horizontal scroll sections | Undiscoverable, frustrating |
| Auto-play video with sound | Instant bounce |
| Parallax on every section | Motion sickness, performance |
| Modal popups on page load | Trust destroyer |
| Carousel for key content | Low engagement, hides content |
| CTA entrance animations | Delays actionability |
| Background music | Never |
| Infinite scroll | Landing pages have an end (the CTA) |

## Performance

| Metric | Target |
|---|---|
| First Contentful Paint | < 1.5s |
| Page weight (excl. images) | < 500KB |
| Largest Contentful Paint | < 2.5s |
| Cumulative Layout Shift | < 0.1 |

- Fonts via `next/font` (self-hosted, subset)
- Images: WebP, `loading="lazy"` below fold, explicit width/height
- Static export (`output: "export"`)
