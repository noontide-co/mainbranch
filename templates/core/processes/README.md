# Processes

Recurring workflows, refined over time.

## Why

Each run refines the process. The playbook gets better.

## Format

```markdown
---
date: 2026-01-13
process: Process Name
frequency: daily | weekly | monthly | as-needed
last_run: 2026-01-13
---

# Process Name

## Trigger
[When to run this]

## Steps
1. Step one
2. Step two
3. Step three

## Checklist
- [ ] Item 1
- [ ] Item 2

## Notes
[Learnings from past runs]
```

## Naming

`process-slug.md` (no date prefix - processes are living docs)

Example: `monthly-bookkeeping.md`

## What Goes Here

- Monthly close
- Weekly reviews
- Content publishing
- Client onboarding
- Anything you do more than once
